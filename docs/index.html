Retry System UberASM

The Retry System is a useful patch for both Kaizo hacks and generally hard hacks, as its primary function is to enable fast level reloading to skip going to the Overworld everytime the player dies (similarly to games like Super Meat Boy).

This package is a spiritual successor of the original Retry patch made by worldpeace, to expand its capabilities and to make it compatible with more resources as a consequence of it being primarily a UberASM resource.

Configuration

The Retry System can be configured in a lot of ways, from basic general settings, to level specific settings, to more nuanced options for advanced users.

All the configuration files you can edit can be found in the retry_config folder and include some comments on how to use them and what they can do, but you can read more in depth explanation for each file below.

The main files you'll need to edit are these:

  • settings_global.asm (Global settings)
  • settings_local.asm (Per-level settings)
  • sram_tables.asm (Save tables)

The other asm files can be ignored for most use cases, but you can check out the Advanced Usage section below for more info.

You can edit the files in the retry_config/code folder (which contains all the source code) to change the retry but this is not recommended unless you know exactly what you're doing.

You can also edit the files in the retry_config/gfx folder to change how the Retry tiles look (see Retry Resources - Graphics for more information).


Multiple Midways

The Retry system also provides for multiple level midways, both as midway objects and automatic midways on level entrance, and many more features. Read how to configure this feature at the following link.

  • Setting up Multiple Midways

Sprite Status Bar

The Retry System comes with a built-in sprite status bar system that can replace the vanilla layer 3 status bar, and includes a minimalistic status bar with display for lives, item box, coins, dragon coins, timer, bonus stars and death counter.

Each of these items can be enabled or disabled at any time, so you can just display what is needed for your hack or for a particular level. This can be useful if you want to use fullscreen layer 3 backgrounds or just want a less dense status bar.

Advanced Usage

Besides the more straightforward global and per-level settings described before, you can use and customize the Retry more in depth if you know your way around coding in asm. Being more niche use cases and more complicated to use, you can avoid it if you're just interested in the run of the mill Retry experience (there's already enough to explore in the other setting files anyway...).

For more information, check out the following pages.

  • prompt_tilemap.asm (Prompt Tilemap Settings): Customization of the Retry prompt text.
  • extra.asm (Extra Routines): Special routines that Retry calls and you can add your code to.
  • ram.asm (Retry RAM Map): Which RAM area Retry uses and how you can interact with it.
  • Retry API Routines: Public Retry routines that you can call from your code.
